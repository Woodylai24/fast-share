# Issue #29: FastShare Folder Access — Explorer Shortcut (PC) + In-App File Browser (Mobile)

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Give users quick access to their received files — PC gets an Explorer shortcut in the titlebar; mobile gets a full in-app file browser with multi-select (batch delete/share), plus image visibility in system Gallery apps.

**Architecture:** PC side is a simple titlebar button calling existing `shell.openPath()`. Mobile side replaces the internal app-private save directory (`getApplicationDocumentsDirectory`) with external app-specific storage (`getExternalFilesDir`) so MediaStore can index images. A new `FileBrowserScreen` reads directly from the filesystem. Migration code moves files from the old location on first launch.

**Tech Stack:** Electron (shell.openPath), Flutter (path_provider, open_filex, url_launcher, dart:io, MethodChannel for MediaScanner)

## Phase 2: Mobile — Migrate Storage to External + Media Visibility

### Task 2: Create storage migration utility

**Objective:** On first launch after update, move files from old internal dir (`<appDocumentsDir>/FastShare/`) to new external dir (`<externalFilesDir>/FastShare/`), then delete the old folder.

**Files:**
- Create: `mobile_client/lib/services/file_storage.dart`

**Step 1: Create `file_storage.dart`**

This is a centralized service for all file storage paths and migration logic.

```dart
import 'dart:io';
import 'package:path_provider/path_provider.dart';
import 'package:shared_preferences/shared_preferences.dart';

class FileStorage {
  static const _migratedKey = 'file_storage_migrated_v2';

  /// Returns the external FastShare directory.
  /// Creates it if it doesn't exist.
  static Future<Directory> getFastShareDir() async {
    final externalDir = await getExternalStorageDirectory();
    if (externalDir == null) {
      // Fallback to internal if external unavailable
      final appDir = await getApplicationDocumentsDirectory();
      return Directory('${appDir.path}/FastShare');
    }
    final fastShareDir = Directory('${externalDir.path}/FastShare');
    if (!await fastShareDir.exists()) {
      await fastShareDir.create(recursive: true);
    }
    return fastShareDir;
  }

  /// Migrate files from old internal storage to new external storage.
  /// Runs once; sets a flag in SharedPreferences to avoid re-running.
  static Future<void> migrateIfNeeded() async {
    final prefs = await SharedPreferences.getInstance();
    if (prefs.getBool(_migratedKey) == true) return;

    try {
      final oldDir = Directory(
        '${(await getApplicationDocumentsDirectory()).path}/FastShare',
      );
      final newDir = await getFastShareDir();

      if (await oldDir.exists()) {
        await for (final entity in oldDir.list()) {
          if (entity is File) {
            final filename = entity.uri.pathSegments.last;
            final newPath = '${newDir.path}/$filename';
            // Don't overwrite if file already exists in new location
            if (!await File(newPath).exists()) {
              await entity.rename(newPath);
            } else {
              await entity.delete();
            }
          }
        }
        // Remove old directory if empty
        if (await oldDir.list().isEmpty) {
          await oldDir.delete();
        }
      }
    } catch (e) {
      // Don't block app launch if migration fails
      print('[FileStorage] Migration failed: $e');
    }

    await prefs.setBool(_migratedKey, true);
  }
}
```

import 'package:flutter/material.dart';
import 'package:fast_share_mobile/services/file_storage.dart';
import 'package:open_filex/open_filex.dart';

class FileBrowserScreen extends StatefulWidget {
  const FileBrowserScreen({super.key});

  @override
  State<FileBrowserScreen> createState() => _FileBrowserScreenState();
}

class _FileBrowserScreenState extends State<FileBrowserScreen> {
  List<FileSystemEntity> _files = [];
  bool _isLoading = true;
  bool _isSelectMode = false;
  final Set<int> _selectedIndices = {};

  @override
  void initState() {
    super.initState();
    _loadFiles();
  }

  Future<void> _loadFiles() async {
    setState(() => _isLoading = true);
    try {
      final dir = await FileStorage.getFastShareDir();
      if (await dir.exists()) {
        final files = await dir.list().toList();
        files.sort((a, b) =>
          FileStat.statSync(b.path)
              .modified
              .compareTo(FileStat.statSync(a.path).modified));
        setState(() {
          _files = files.whereType<File>().toList();
          _isLoading = false;
        });
      } else {
        setState(() => _files = []);
        setState(() => _isLoading = false);
      }
    } catch (e) {
      setState(() {
        _files = [];
        _isLoading = false;
      });
    }
  }

  // ... continued in next tasks
}

    setState(() {
      _isSelectMode = true;
      if (_selectedIndices.contains(index)) {

        if (_selectedIndices.isEmpty) _isSelectMode = false;

## Risks & Tradeoffs

1. **MediaStore visibility on Android 11+:** `getExternalFilesDir()` returns `/storage/emulated/0/Android/data/<package>/files/`. On Android 11+, MediaStore *should* index media from this location when `MediaScannerConnection.scanFile()` is called, but some OEMs may skip it. If this becomes an issue, the fallback is to save images to a public directory (`Pictures/FastShare/`) using the MediaStore API directly, which requires more complex code.

2. **Migration race condition:** The migration runs once on app start. If the user receives a file while migration is running (unlikely but possible), the save path might not be ready. Mitigated by `FileStorage.getFastShareDir()` creating the directory on every call.

3. **`share_plus` for batch share:** The `shareXFiles` method uses content URIs under the hood. On some Android versions, sharing to certain apps may fail if those apps can't handle content URIs. This is a known limitation of the share mechanism, not specific to this implementation.

4. **`getExternalStorageDirectory()` behavior:** On some Android devices, this may return null (rare, but possible on devices without external storage). The fallback in `FileStorage.getFastShareDir()` handles this by falling back to internal storage.
